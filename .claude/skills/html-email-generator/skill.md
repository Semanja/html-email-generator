# html-email-generator

Конвертер `.docx` → HTML для email-рассылок. Офлайн-инструмент, работает в браузере.

## Структура

```
html-email-generator/
├── html_generator.html       ← главный файл, открывается в браузере
├── инструкция.html           ← подробная инструкция с FAQ
├── mammoth.browser.min.js    ← библиотека Mammoth для парсинга .docx
├── README.md                 ← ридми проекта
├── make_html_archive.py      ← скрипт сборки архива для передачи
└── .claude/skills/html-email-generator/skill.md  ← этот файл
```

## Как запустить

Просто открыть `html_generator.html` в браузере — Chrome, Firefox, Edge, Safari.

Никакой сервер, установка зависимостей или интернет не нужны. Проект полностью статический.

## Как работает

1. Пользователь перетаскивает `.docx`-файл в зону загрузки
2. Нажимает «⚙️ Конвертировать»
3. Mammoth парсит Word-документ в HTML
4. Если в тексте есть строки `Кнопка: Текст надписи` — открывается модалка для настройки цвета и URL
5. Готовый HTML применяет инлайн-стили для email-совместимости (Gmail, Unisender, GetCourse и т.д.)
6. Пользователь копирует HTML кнопкой «📋 Скопировать»

## Ключевые моменты

### Mammoth
- Файл `mammoth.browser.min.js` должен лежать рядом с `html_generator.html`
- Если его нет — показывается предупреждение со ссылкой на скачивание
- `styleMap` мапит Word-стили заголовков, подчёркивание и зачёркивание
- Изображения конвертируются в base64 и встраиваются прямо в HTML

### Кнопки
- Строка `Кнопка: Текст надписи` в Word → кнопка в HTML
- Модалка: цвет фона, цвет текста, ссылка
- Можно пропустить — тогда останется как обычный текст
- Кнопок может быть несколько — показываются по очереди

### Инлайн-стили
- Функция `toEmailHtml()` через DOMParser расставляет стили на все теги
- Таблицы получают рамки и отступы (mammoth их не проставляет)
- Всё обёрнуто в `<div style="max-width:600px">` — стандарт для email

### Ограничения
- `.doc` не поддерживается браузером — нужно пересохранить как `.docx`
- Крупные изображения раздувают base64 — если рассылка лимитирует размер, нужно сжимать картинки в Word

## Инструкция
- Файл `инструкция.html` содержит полное руководство и FAQ
- Ссылка на него есть в шапке генератора
- При редактировании нужно синхронизировать изменения между `инструкция.html` и генератором

## Публикация

Проект — статический HTML, можно разместить где угодно:
- **GitHub Pages** — просто залить файлы в репозиторий
- **Vercel / Netlify** — перетащить папку
- **Локально** — раздать архивом

### Сборка архива

```bash
cd html-email-generator
python make_html_archive.py
```

Создаст `html-email-generator-YYYY-MM-DD.zip` в корне проекта.

## Типовые изменения

### Добавить / убрать элемент в HTML
1. Отредактировать `html_generator.html` — CSS, HTML-разметку, JavaScript
2. Если меняется логика конвертации — проверить что все paths (`.docx`, mammoth) остаются валидными
3. Если меняется инструкция — отредактировать `инструкция.html`
4. Обновить `README.md`

### Обновить инлайн-стили
В объекте `STYLE` внутри функции `toEmailHtml()` — каждая запись добавляет или заменяет стиль для тега.

### Добавить новый HTML-тег в конвертацию
Добавить запись в объект `STYLE` и, если надо, в `styleMap` Mammoth (для Word-стилей).

### Обновить mammoth
Скачать свежую версию с CDN и заменить `mammoth.browser.min.js`:
```
https://cdn.jsdelivr.net/npm/mammoth/mammoth.browser.min.js
```

## Проверка перед публикацией

- [ ] `mammoth.browser.min.js` лежит рядом с `html_generator.html`
- [ ] Генератор открывается локально (двойной клик)
- [ ] Конвертация `.docx` в HTML работает (тестовый документ с заголовками, таблицей, списком, картинкой, кнопкой)
- [ ] Кнопка «📋 Скопировать» копирует HTML
- [ ] Полученный HTML вставляется в Unisender / GetCourse, письмо открывается в Gmail
- [ ] `инструкция.html` открывается и соответствует текущей версии генератора
- [ ] README.md актуален
- [ ] `make_html_archive.py` собирает архив без лишних файлов
